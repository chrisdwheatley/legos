Button

When creating a button please be sure to use a <button> or <a> tag for accessibility purposes. There are three different types of buttons.

Attributes:
  • type : "submit", "reset", "button"
  • aria-label : defines a label for the element (only used when label text not visible on screen)
  • name
  • value

Skin

Buttons have their own specific skin styles. They are very similar to the global skins but with additional styles just for them.

  • btn--{{ color }}
  • btn--outline-{{ color }}

Filled

Filled buttons have colored borders and backgrounds with white font color.

Demo:
Markup:
<button type="button" class="btn btn--blue">...</button>
<button type="button" class="btn btn--green">...</button>

Outlined

Outlined buttons only have colored text and borders, otherwise, they are the same as filled buttons. The same rules apply.

Demo:
Markup:
<button type="button" class="btn btn--outline-red">...</button>
<button type="button" class="btn btn--outline-orange">...</button>

Links

This element is not a <button> element, it is infact an <a> element.

While not specifically a button (it does not get the .btn base) links are still within the button category. They get the same skins and the same type of hover actions but do not take up the same amount of space. A link takes up only the amount of space it needs for what's inside of it, which is unlike buttons that have set padding.

Links do not have outline skins like buttons do.

*Note: Anchor tags do not have a :disabled state. Use the aria-disabled="true" attribute to style a disabled anchor element the same way a disabled button is styles.

Demo:
link {{ color }} disabled link {{ color }}
Markup:
<a class="link--blue">...</a>
<a class="link--green" aria-disabled="true">...</a>

Block Buttons

Block buttons span the entire width of their parent container.

Demo:
Markup:
<button type="button" class="btn btn--block">Default Size</button>
