buttons

When making a button please be sure to use a button or a tag for accessability purposes. There are three different type of buttons.

Filled

Filled buttons have colored borders and backgrounds with a white font color.

Demo:
Markup:
<button class="btn btn--default">...</button>

Outlined

Outlined buttons only have colored text and borders, otherwise they are the exact same as filled buttons. Same rules apply.

Demo:
Markup:
<button class="btn btn--outline-default">...</button>

Links

This element is not a <button> tag, it is infact <a> tag.

While not specifically a button, it does not get the .btn base, links are still within the button category. They get the same skins and the same type of hover actions but do not take up the same amount of space. The link space takes up only the amount of what's inside of it, which is unlike buttons that have a set padding.

Demo:
link primary / default link success link danger link gray
link white

disabled link primary / default disabled link success disabled link danger disabled link gray
link white
Markup:
<a class="link--default">...</a>

Colors

  • --primary
  • --success
  • --danger
  • --default

Block Buttons

For block buttons that span the entire width of the parent container use .btn--block.

Demo:
Markup:
<button class="btn btn--danger btn--block">Default Size</button>
