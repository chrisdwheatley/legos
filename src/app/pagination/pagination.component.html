pagination

We have default pagination. We should have a few different skins to choose from that we can put on top of these base styles.

Labelling the pagination component:
For accesability purposes the pagination component should be wrapped in a <nav> element to identify it as a navigation section to screen readers and other assistive technologies. It's also possible that a page may have more than one <nav> section already (such as the primary navigation in the header, or a sidebar navigation). It would be good to follow best practices and provide a descriptive aria-label for the <nav>.

Example: If the pagination component is used to navigate between a set of search results, an appropriate label could be aria-label="Search results pages".

Both show a disabled button on the first item and the active state on the the 1 button.

The first is a baseline example of the default pagination.
