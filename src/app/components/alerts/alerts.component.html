alert

Normally alert banners are global notifications that are displayed as a banner at the top of a user's screen. They are used to make the user aware of a state change (like the system being down, planned maintenance, or a new feature, etc).

Alert banners get fixed to the top of the screen and usually stay till the user dismisses. This is what differentiates it from a toast element that fades away after a certain amount of time on screen.

Attributes:
  • role : 'alert'
  • aria-label : add to the close button, it is a button without content inside
Demo:
This is the default alert banner

For demonstration purposes, this alert banner is inside the document for you to see what it looks like instead of fixing it to the top of the screen where it would be. At its core, it is just a rectangle till you add a skin indicating what type of alert it is.

Markup:
<div class="alert" role="alert">
  <div class="alert__body">Alert text here</div>
</div>

Close (optional)

In certain cases you might want the user to be able to dismiss the alert, acknowledging that they saw it. To do this just add a close button inside the alert after the alert__body.

Demo:
An alert banner with an optional close button
Markup:
<div class="alert role="alert">
  <div class="alert__body">Alert text here</div>
  <button type="button" aria-label="alert close button" class="alert__close">
    <i class="fas fa-times close" aria-hidden="true"></i>
  </button>
</div>

Skins

  • alert--danger
  • alert--success
  • alert--primary
  • alert--warning

The skin color controls:

  • Backgroud Color
  • Font Color
  • Close Button Color (optional add-on)
  • Border Color (optional add-on)

Border

If you would like to add a border around your alert add the --border modifier to the skin class.
