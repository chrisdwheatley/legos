pagination

For showing multiple pages of similar content.

Attribute: aria-label="", role="menu"

Accesability

For accesability purposes the pagination component should be wrapped in a <nav> element to identify it as a navigation section. It is also possible and highly likely that a page may have more than one <nav> element already (such as the primary navigation in the header, or a sidebar navigation). Best practices dictate providing a descriptive aria-label for the <nav> element to signal to screen readers and other assistive technologies what the element really is.

Example: If the pagination component is used to navigate between a set of pages in a sign up form, an appropriate label could be aria-label="sign up form pages".

    Prev/Next Labels

    You can customize how you want to represent the previous/first and next/last items. You could choose to have all of them displayed, or just choose one set. Either way you can use words or symbols. The only thing that changes when you add --first or --last to .pagination__item is the spacing. The first and last buttons have a margin that seperate them from the inner set of buttons.
