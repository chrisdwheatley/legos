Toast

Toasts are dynamic, they are displayed for a short period and typically contain system feedback concerning a user's interaction. It isn't always and is not necessarily feedback of a user action. The content shown in the messages it can be helpful to the user but not as important to break the users' current workflow and divert their complete attention to that message.

Example: You added something to your cart or you updated your password successfully.

Attributes:
  • role : 'toast'
Demo:
This is the default toast
Markup:
<div class="toast" role="toast">
  <div class="toast__body">Toast text here</div>
</div>

Close (optional)

In certain cases you might want the user to be able to dismiss the toast, acknowledging that they saw it. Add a close button inside the toast, either inside the toast__header or the toast__body.

Demo:
Toast Header
A toast with an optional dismiss
Markup:
<div class="toast" role="toast">
  <div class="toast__header">
    Toast Header
    <button type="button" class="toast__close">
      <i class="icon fas fa-times" aria-hidden="true"></i>
    </button>
  </div>
  <div class="toast__body">A toast with an optional dismiss</div>
</div>
