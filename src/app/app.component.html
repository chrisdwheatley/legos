Naming Convention: BEM

The primary purpose of BEM naming convention is to give names meaning so that they are as informative as possible for the developer.

BEM stands for:

  • B: Block
  • E: Element
  • M: Modifier
Rules:
  • Names are written in lowercase letters.
  • Words are separated by a hyphen (-).
  • If a section of code can't be used separately without the parent entity (the block).
Block

The block name defines the namespace for its elements and modifiers. Think of it as the parent.

Element

The element name is separated from the block name by a double underscore (__). These are elements that are inside the parent that rely on it.

Modifier

A modifier is an entity that defines the appearance, state, or behavior of a block or element. The modifier name is separated from the block or element name by two hyphens (--). Modifiers can manipulate the block or element so in order to theme or style that particular component without influincing the styles of the original component.
