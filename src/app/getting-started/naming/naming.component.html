naming convention: BEM

The LegoCSS library uses the BEM naming convention. The primary purpose is to give names meaning so that they are as informative as possible for the developer. For a small refresher see below.

B: Block

The block name defines the namespace for its elements and modifiers. This is the parent element.

.block
E: Element

The element name is separated from the block name by a double underscore ( __ ). These are elements that are inside the parent that relies specifically on it.

.block__element
M: Modifier

A modifier defines the appearance, state, or behavior of a block or element. The modifier name is separated from the block or element name by two hyphens ( -- ). Modifiers can manipulate the block or element to theme or style that particular component without influencing the styles of the original component.

.block__element--modifier

Naming Rules:

  • Names are written in lowercase letters
  • Words are separated by a hyphen (-)
