Accessibility

In compliance with Web Content Accessibility Guidelines (WCAG), 2.0 our components strive to be completely accessible to assistive technologies as well as promoting best practices for developers.

ARIA: Roles, states, and properties

Being ARIA compliant for assistive technologies has become the standard. This includes having the correct value for the role, and aria-label attributes alongside the standard element attributes, id, for, type, title, name, and value.

Images

Always provide alternative text for images. The alt attribute is not only used for assistive technologies like screen readers. It is also useful if a user has a slow internet connection and cannot view the image or if there is an error finding the image.

<img src="cat.png" alt="A picture of a orange cat sleeping on a brown chair">
Colors

Colors are a very important part of a website. There are several different impairments that a user can have. We as designers need to accommodate those who are blind, color blind, and or have partial impairments. It is also best practice to ensure the contrast between text and background colors are enough for those with color blindness. There are several free tools available that will scan your page and compare text and background.

  • Colorblind Web Page Filter - examples of different levels of colorblindness
  • Color Contrast Accessibility Validator - scans websites to verify the contrast and ensure better accessibility
Links

Just like with images it is important to add an alt attribute on anchor elements that link to an outside website or file, describing what it is. If you are linking to a document, for example, a PDF, say that in the description. Users want to know what they will be opening ahead of time. Always check your links before your website goes live and make sure they are working and going where they are supposed to. You wouldn't want a user to click a link that goes nowhere.

<a href="http://wwww.gosomewhere.com" alt="A link to a new page" rel="noreferrer">
