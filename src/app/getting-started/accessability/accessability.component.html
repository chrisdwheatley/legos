accessability

In compliance with Web Content Accessibility Guidelines (WCAG) 2.0 our components strive to be completely accessible to assistive technologies as well as promting best practices for developers.

Images

Always provide alternative text for images. The alt attribute is not only userful for assistive technologies like screen readers. It is also useful if the user has a slow internet connection and cannot view the image, if there is an error or in the src.

<img src="cat.png" alt="A picture of a orange cat sleeping on a brown chair">
Colors

Colors are a very important part of a website design. There are several different imparements that a user can have. We need to not only accomdate those who are blind but those who have partial imparements, and different types of color blindness. Apart from visual imparenments it is also best practice to endure that there is a good amount of contrast between the text and background color. There are several free tools availible that will scan your page and compare text and background colors.

  • Colorblind Web Page Filter - examples of different levels of colorblindness
  • Color Contrast Accessibility Validator - scans websites to verify the contrast and ensure better accessibility
Links

Just like with images it is important to add a of where the link goes. If you are linking to a documenet, for example a PDF, be sure to say that in the description. Users want to know what they will be opening ahead of time. Best practie always dictates that you check all your links before your site goes live to make sure they are all working. You would not want a user to click a dead link that goes no where.

<a href="http://wwww.gosomewhere.com" alt="A link to a new page" rel="noreferrer">
