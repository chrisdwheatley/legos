accessability

In compliance with Web Content Accessibility Guidelines (WCAG) 2.0 our components strive to be completely accessible to assistive technologies as well as promting best practices for developers.

Images

Always provide alternative text for images. The alt attribute is not only userful for assistive technologies like screen readers. It is also useful if the user has a slow internet connection and cannot view the image, if there is an error or in the src.

<img src="cat.png" alt="A picture of a orange cat sleeping on a brown chair">
Colors

Colors are very important on a website. There are several different sight imparements that a user can have. We need to not only accomdate those who are blind but those who have partial blindness as well as other sight imparements. Check out this demo for examples on how some are affected. Ensure that there is a high contrast between the text and background color. There are several free tools availible that will scan your page and compare text and background colors.

Text

It is important that text can be scaled up and down without affecting the rest of page content.

Links

Just like with images it is important to add descriptions such as what the links are or where they go to. If you are linking to a documenet, for example a PDF, be sure to say that in the description. Users want to know what they will be opening ahead of time. Best practie always dictates that you check all your links before your site goes live to make sure they are all working. You would not want a user to click a dead link that goes no where.
