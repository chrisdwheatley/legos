Accessibility

In compliance with Web Content Accessibility Guidelines (WCAG), 2.0 our components strive to be completely accessible to assistive technologies as well as promoting best practices for developers.

ARIA: Roles, states, and properties

Being ARIA compliant for assistive technologies has become the standard. This includes having the corrent value for role, aria-label attributes along side standard element attributes like id, for, type, title, name, value.

Images

Always provide alternative text for images. The alt attribute is not the only use for assistive technologies like screen readers. It is also useful if the user has a slow internet connection and cannot view the image, if there is an error or in the src.

<img src="cat.png" alt="A picture of a orange cat sleeping on a brown chair">
Colors

Colors are a very important part of website design. There are several different impairments that a user can have. We need to not only accommodate those who are blind but those who have partial impairments, and different types of color blindness. Apart from visual impairments, it is also best practice to endure that there is a good amount of contrast between the text and background color. There are several free tools available that will scan your page and compare text and background colors.

  • Colorblind Web Page Filter - examples of different levels of colorblindness
  • Color Contrast Accessibility Validator - scans websites to verify the contrast and ensure better accessibility
Links

Just like with images it is important to add an alt attribute where the link goes. If you are linking to a document, for example, a PDF, be sure to say that in the description. Users want to know what they will be opening ahead of time. Best practice always dictates that you check all your links before your site goes live to make sure they are all working. You would not want a user to click a dead link that goes nowhere.

<a href="http://wwww.gosomewhere.com" alt="A link to a new page" rel="noreferrer">
